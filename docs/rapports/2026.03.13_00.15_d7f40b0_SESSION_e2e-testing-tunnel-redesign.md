## Decisions
1. **Hydra headed mode for E2E**
   - **Choix**: Close headless, create headed instance
   - **Rationale**: Cloudflare blocks headless on claude.ai consistently
   
2. **Always-on logging (no debug flag)**
   - **Choix**: Replace `debug: true` flag with always-on RollingLogger capped at 2MB
   - **Rationale**: User asked for "autant de logs que nécessaire" — debug flag was hiding critical info

3. **Integrated Cloudflare Tunnel in tray**
   - **Choix**: Spawn `cloudflared` from Electron, auto-extract URL, build share link with token
   - **Rationale**: "Partager doit être simple" — one click instead of manual terminal + URL construction

4. **Viewer theme: Claude warm sand**
   - **Choix**: Custom CSS with CSS variables, warm beige/sand tones
   - **Rationale**: Match Claude's actual UI identity rather than generic blue/white

## Problems
1. **interactive_shell path mangling**
   - **Symptôme**: Paths with spaces break in interactive_shell command
   - **Solution**: Use `cwd` parameter instead of `cd` in command

2. **Empty fullSync on WS connect**
   - **Symptôme**: Viewer shows "Waiting for conversation" despite extension connected
   - **Cause**: `fullSync()` called in `ws.onopen` before React renders the DOM
   - **Solution**: Check `getMessageContainer()` first, defer to `startObserving` if null

3. **Token auth blocking Electron viewer**
   - **Symptôme**: Viewer connected but never received messages
   - **Cause**: `TOKEN = true` in main.js auto-generates token, but viewer loaded via `file://` had no `?token=` param
   - **Solution**: Made `getWsUrl()` async, fetches token via `window.liveShare.getToken()` IPC first

4. **Cloudflare Tunnel Error 1033**
   - **Symptôme**: Old tunnel URL returns error after restart
   - **Cause**: Quick tunnels get new URLs on each restart
   - **Solution**: Documented behavior, new tunnel = new URL
