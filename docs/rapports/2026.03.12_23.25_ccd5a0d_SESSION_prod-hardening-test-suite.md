## Decisions
1. **Token auth for viewers only, not sources**
   - **Choix**: Sources (extension) connect without token
   - **Rationale**: Extension runs on localhost only, adding token would complicate extension config
   - **Alternative rejetée**: Mutual auth — unnecessary complexity for localhost-only source

2. **Unified viewer file**
   - **Choix**: Delete `server/public/index.html`, use `viewer/index.html` everywhere
   - **Rationale**: Electron-specific features (`webkit-app-region`, `window.liveShare`) are already guarded by conditionals
   - **Alternative rejetée**: Keep both files synced — guaranteed to drift

3. **Delta optimization: append detection**
   - **Choix**: Compare new text with cached, send `delta` (append) if text was only extended
   - **Rationale**: Reduces bandwidth during streaming (most updates are appends)
   - **Alternative rejetée**: Binary diff — overkill for this use case

4. **Hydra headed mode for E2E validation**
   - **Choix**: Close headless instance, create headed instance to bypass Cloudflare
   - **Rationale**: Cloudflare blocks headless browsers consistently on claude.ai
   - **Alternative rejetée**: Headless with extra stealth flags — unreliable

## Problems
1. **Interactive shell path mangling**
   - **Symptôme**: `interactive_shell` with `cd "path with spaces" && python script.py` failed with "syntaxe incorrecte"
   - **Cause**: Git Bash path handling in interactive_shell doesn't handle spaces well
   - **Solution**: Use `cwd` parameter instead of `cd` in command
   - **Prévention**: Always use `cwd` parameter for interactive_shell when paths have spaces

2. **Cloudflare blocking Hydra on claude.ai**
   - **Symptôme**: "Just a moment..." page, Cloudflare challenge checkbox
   - **Cause**: Headless browser detection
   - **Solution**: Created headed (visible) browser instance — Cloudflare auto-solved
   - **Prévention**: Always use `headless: false` for Cloudflare-protected sites

3. **Test WebSocket message buffering**
   - **Symptôme**: First test timed out waiting for full_sync message
   - **Cause**: Server sends full_sync immediately on viewer connect, before `waitForMessage` listener attached
   - **Solution**: Added message queue buffer in `connectWs` — captures messages arriving before anyone calls `waitForMessage`
   - **Prévention**: Always buffer messages from connection start in WS test helpers
